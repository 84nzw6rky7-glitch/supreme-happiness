## Product direction

This project supports a safer social/community experience for younger groups, families, and educators.

### Core safety and trust requirements

- **No permanent data storage by default**: user-generated and account activity data should use minimal-retention handling, with clear deletion windows and a user-facing privacy notice.
- **Account contact requirement**: each account must include at least one verified contact method:
  - phone number, or
  - email address.
- **Harassment protection**: users must be able to block accounts that harass them.
- **Clear capability indicator**: blocking/reporting availability must be obvious in the UI (for example, a visible "Block user" and "Report user" action in profiles and chats).
- **Anti-scam warning**: show a clear, proactive safety message (popup/banner) that reminds users to protect themselves from fake accounts and scams while using the internet.

### Subscription and monetization

- **Target subscription price**: `$14.99/month`.
- **Merchandise support**: include a merchandise area/storefront for branded products.

### Content/media support

- Images provided for launch are mostly ready, with light refinement/touch-up workflows still needed before final publishing.

### Audience and dashboard experience

- Support younger user groups with age-appropriate interactions and safety defaults.
- Provide teacher-parent communication and interaction tools.
- Include dashboards for specific topics (for example: safety updates, student/community activity, and communication summaries).
